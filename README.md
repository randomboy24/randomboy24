## The Kind of Systems I Like Building

```txt
User action
   ↓
API layer
   ↓
Validation + auth
   ↓
Database transaction
   ↓
Queue / worker
   ↓
External API or AI tool call
   ↓
State update
   ↓
Retry / failure handling
   ↓
User-facing result
```

I care about what happens inside the system: data flow, failure handling, async execution, observability, and making the product reliable enough for real users.
